### Dataset Generation Methods

**LLM (tenth-trial)**  
- Dirty data generated by a large language model given the clean record + injection instructions  
- Errors include realistic value substitutions, plausible but wrong categorisations, obfuscated strings  
- High error rate (~22% of cells) due to the generative nature of the injection

**Mixed SOTA (Adult Income)**  
- **Phase 1 — Unintentional:** `tab_err` (Biessmann et al.) — Typo, MissingValue, Outlier, CategorySwap under ECAR/EAR/ENAR mechanisms (7,877 cells)  
- **Phase 2 — Intentional:** Kireev Universal Greedy Search adversarial attack, goal: flip prediction ≤50K → >50K, cost bound 20.0/row (3,838 cells)  
- Mask encoding: `-1` = unintentional, `0` = clean, `+1` = intentional (last-writer-wins)

**Mixed SOTA (TwiBot-20)**  
- **Phase 1 — Unintentional:** `tab_err` Outlier injection (all features are numeric integers), ECAR (756 cells)  
- **Phase 2 — Intentional:** Fast plausible adversarial attack (±1σ, clamped to [p5, p95]), goal: flip bot (1) → human (0), cost bound 30.0/row (346 cells)  
- 8 active numeric features: `followers_count`, `friends_count`, `statuses_count`, `listed_count`, `verified`, `protected`, `description_length`, `name_length`

## Key Observations

### 1. The pipeline generalises across datasets and domains
All three datasets — different domains (income vs. bot detection), different sizes (1K–43K errors), different injection strategies — achieve AUC > 0.97 and F1 > 0.90. The 13 heuristic features carry sufficient discriminative signal regardless of context.

### 2. Mixed SOTA (Adult) is the easiest to classify (F1 = 0.984)
The adversarial Kireev attack produces very clean signal: intentional errors are **always** plausible (`h1_plausible = 1.0`) and never obfuscated (`h2_is_obfuscation ≈ 0`). Unintentional `tab_err` noise is often non-plausible and frequently obfuscated. This polarity makes the two classes nearly linearly separable.

### 3. LLM errors are the hardest to classify (F1 = 0.903)
LLM injections are semantically rich — plausible but not always maximally so (0.76 vs. 1.0), partially obfuscated (0.23), and with only moderate distribution shift (h3 delta = +0.55). The classes overlap more than in the algorithmic datasets, which is reflected in the higher error counts and lower precision.

### 4. TwiBot-20 shows higher variance (Std F1 = 0.030)
Small dataset (1,099 erroneous cells) amplifies fold-to-fold variance. H2 features (`h2_min_edit_distance`, `h2_is_obfuscation`) contribute zero importance — they are text-based heuristics and the twitter features are all numeric integers. The pipeline still achieves F1 = 0.903 / AUC = 0.980 driven entirely by H1, H3, H4, H5, H6, H7.

### 5. Feature importance shifts with injection strategy

| Feature | LLM | Mixed Adult | TwiBot |
|---------|-----|-------------|--------|
| **h3_distribution_score** | #1 (0.259) | #3 (0.136) | #2 (0.176) |
| **h4_coherence_score** | #2 (0.183) | #7 (0.058) | #3 (0.175) |
| **h1_plausible** | #3 (0.180) | #2 (0.165) | #1 (0.206) |
| **h7_gain_direction** | #6 (0.055) | **#1 (0.213)** | #7 (0.070) |
| **h2_is_obfuscation** | #4 (0.138) | #6 (0.089) | — (0.000) |
| **h5_codependent_flag** | low | low | #4 (0.139) |

- **H7 gain direction** dominates for Kireev adversarial (Adult): expected — the attack is specifically designed to flip the prediction, meaning it always changes features in the "gain" direction.  
- **H2 obfuscation** matters for LLM: LLM sometimes introduces subtly misspelled or encoded strings.  
- **H5 codependent** matters for TwiBot: the pipeline detects correlated feature changes within a row (e.g. `followers_count` and `friends_count` both changed together by the adversarial attack).
